## 🚀 שלב 2: העלאה ל-Vercel

Vercel הוא השירות שיציג את האתר באוויר. כבר נרשמת אליו בתחילת התהליך.

### 2.1. כניסה ל-Vercel
1. היכנסי ל-**vercel.com** והתחברי.
2. בדאשבורד הראשי תראי כפתור **"Add New..."** או **"+"** למעלה.
3. לחצי עליו → בחרי **"Project"**.

### 2.2. בחירת אופציית העלאה
תראי שתי אופציות:
- **Import Git Repository** — לא רלוונטי לנו עכשיו.
- **Deploy directly via CLI** — לא רלוונטי.

⚠️ **רגע — Vercel לא מציג ברירת מחדל אופציה לגרירת קבצים.** במקום זאת, נשתמש ב-**GitHub** כפי שהתכוננו לעשות.

### 2.3 (חלופה פשוטה יותר): שימוש ב-Drop של Vercel

Vercel תומך בהעלאה ישירה דרך ממשק נפרד:
1. היכנסי ל-**[vercel.com/new](https://vercel.com/new)**.
2. גללי למטה — תראי קישור: **"Try a Template"** או **"Deploy without Git"**.

**אם לא מצאת — האפשרות הבטוחה ביותר היא דרך GitHub. ראי שלב 2.4.**

### 2.4. העלאה דרך GitHub (הדרך המומלצת)

#### 2.4.1. יצירת Repository ב-GitHub
1. היכנסי ל-**github.com** והתחברי.
2. לחצי **"+"** בפינה הימנית-עליונה → **"New repository"**.
3. בשדה **"Repository name"** — הקלידי: `alonaphoto`.
4. השאירי **Public**.
5. **אל תסמני** "Add a README".
6. לחצי **"Create repository"**.

#### 2.4.2. העלאת קבצים ל-Repository
1. בעמוד החדש שנפתח, לחצי על **"uploading an existing file"** (קישור באמצע העמוד).
2. **גררי את כל התיקייה `website/`** מהמחשב לתוך החלון.
   - או לחצי **"choose your files"** ובחרי הכל.
3. חכי שהקבצים יעלו (כדקה — בגלל התמונות).
4. למטה, בשדה **"Commit changes"** — אפשר להשאיר את הברירת מחדל.
5. לחצי **"Commit changes"**.

#### 2.4.3. חיבור ל-Vercel
1. חזרי ל-**vercel.com** → **"Add New..."** → **"Project"**.
2. תראי רשימה של ה-repositories שלך מ-GitHub. **אם לא רואה** — לחצי **"Adjust GitHub App Permissions"** ואשרי גישה.
3. ליד `alonaphoto` — לחצי **"Import"**.
4. במסך ההגדרות:
   - **Project Name:** `alonaphoto`
   - **Framework Preset:** *Other* (זיהוי אוטומטי)
   - **Root Directory:** השאירי כברירת מחדל
   - שאר ההגדרות — בלי לגעת.
5. לחצי **"Deploy"**.
6. חכי 30 שניות. תראי הודעה: **"Congratulations! 🎉"**.

✅ **האתר עכשיו באוויר!** הכתובת תיראה משהו כמו `alonaphoto-xyz123.vercel.app`.

## 🌐 שלב 3: חיבור הדומיין alonaphoto.co.il

עכשיו נחבר את הדומיין שקנית.

### 3.1. הוספת הדומיין ב-Vercel
1. בפרויקט שלך ב-Vercel → לחצי **"Settings"** למעלה.
2. בתפריט הצד — **"Domains"**.
3. בשדה ההזנה — הקלידי: `alonaphoto.co.il`.
4. לחצי **"Add"**.
5. Vercel יציג לך **2 רשומות DNS** שצריך להוסיף ב-Cloudflare:
   ```
   Type: A     Name: @     Value: 76.76.21.21
   Type: CNAME Name: www   Value: cname.vercel-dns.com
   ```
   (הערכים המדויקים יוצגו לך — תעתיקי אותם משם!)

### 3.2. הוספת הרשומות ב-Cloudflare
1. היכנסי ל-**cloudflare.com** → לחצי על `alonaphoto.co.il`.
2. בתפריט הצד → **"DNS"** → **"Records"**.
3. **רשומה 1 (A record):**
   - לחצי **"+ Add record"**.
   - Type: **A**
   - Name: **@**
   - IPv4 address: הערך מ-Vercel (76.76.21.21 או דומה)
   - Proxy status: **DNS only** (אפור, לא כתום!)
   - TTL: **Auto**
   - **Save**.
4. **רשומה 2 (CNAME record):**
   - לחצי שוב **"+ Add record"**.
   - Type: **CNAME**
   - Name: **www**
   - Target: הערך מ-Vercel (cname.vercel-dns.com)
   - Proxy status: **DNS only**
   - TTL: **Auto**
   - **Save**.

### 3.3. אישור ב-Vercel
1. חזרי ל-Vercel → דף ה-Domains של הפרויקט.
2. תראי את הסטטוס משתנה תוך 1-5 דקות מ-**"Invalid Configuration"** ל-**"Valid Configuration"** ✅.
3. SSL (HTTPS) יוגדר אוטומטית תוך כמה דקות.

### 3.4. בדיקה
היכנסי ל-`alonaphoto.co.il` בדפדפן. **האתר שלך באוויר!** 🎉

## 📝 שלב 4: עדכון תוכן בעתיד

### איך מוסיפים אירוע חדש?

1. **פתחי את `index.html`** ב-Notepad / VS Code / כל עורך טקסט.
2. **חפשי את השורה** `const EVENTS = [` (בערך באמצע הקובץ).
3. **הוסיפי אירוע חדש** באותו פורמט:

```javascript
{
  id: 'event_chadash',     // מזהה ייחודי באנגלית, ללא רווחים
  name: 'שם האירוע',
  date: '15.05.2026',
  location: 'מיקום',
  cover: 'photos/event4_cover.jpg',
  new: true,
  riders: [
    { number: '50', bike: 'KTM', photos: ['event4_rider50_1.jpg', 'event4_rider50_2.jpg'] }
  ]
}
```

4. **העלי את התמונות** לתיקיית `photos/`.
5. **שמרי את `index.html`** ועלי את השינויים ל-GitHub:
   - היכנסי ל-Repository ב-GitHub.
   - **Add file** → **Upload files** → גררי את הקבצים החדשים.
   - **Commit changes**.
6. Vercel יזהה את השינוי ויעדכן את האתר תוך **30 שניות**. אוטומטית.

### איך מוחקים אירוע (אחרי 14 יום)?

פשוט — מחקי את הבלוק שלו מ-`EVENTS` בקובץ `index.html`, וגם את התמונות שלו מתיקיית `photos/`. שמרי, העלי ל-GitHub.
